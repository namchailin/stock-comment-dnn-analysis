# 주식 댓글로 "실력자"를 감지할 수 있는가 — 한국어 댓글 4-Class 분류기

> **연구 질문**
> 1. 댓글로 주가를 맞힌 사람들 중에 **진짜 실력자**가 있는가?
> 2. 그 실력자를(즉 *적중하는 댓글을*) **모델로 감지**할 수 있는가?
>
> 토스증권 TSLA·NVDA 댓글을 **4-Class**(예측없음 / 예측실패 / 방향적중 / 날짜·방향적중)로 라벨링하고,
> 비용 한도 내에서 **희소클래스 문제를 피해** 분류기를 학습해 *적중 댓글 감지 성능*을 검증한다.
> (적중이 운인지 실력인지는 실력점수 일관성 지표로 **가볍게** 곁들인다 — 부차적.)

---

## 4-Class 라벨 정의
| Class | 의미 |
|---|---|
| **C0** | 예측 없음 (잡담·리액션) |
| **C1** | 예측했으나 실패 |
| **C2** | 방향 적중 |
| **C3** | 날짜 + 방향 적중 |

> 핵심 설계: **LLM은 댓글에서 예측만 추출**(시점·방향·날짜·확신도·근거), **적중 여부는 우리 코드가 주가로 결정**적으로 계산한다 → LLM에 미래정보 노출 0 (누수 방어).

---

## 파이프라인 (폴더 = 스토리 순서)
```
1_collect/        크롤링 + HMAC 익명화 + 주가 수집
2_eda/            탐색 → 변동 판정선(h=5,k=1.0)·max_len=512 확정
3_label/          LLM 4-Class 라벨링 (프롬프트·가격맥락·러너) + 주가로 Class 확정
4_train1_verify/  ── Stage 1 ── 베이스라인 분류기: 적중(C2/C3) 감지 신호가 있나?
                  (learning_curve로 "데이터 늘리면 좋아지나" 진단 → 보강 근거)
5_train2_retrain/ ── Stage 2 ── 2차 라벨로 희소클래스 보강 → 재학습, 감지 성능↑
6_skill_check/    (보조·가벼움) 실력점수 일관성으로 운/실력 가볍게 유추
lib/              공유 유틸(exploration: 시각화·주가로더 / train_colab: 학습엔진)
data/             labeled/(익명화 완료) · unlabeled/{raw[비공개], splits}
results/figures/  그림
archive/          일회성·심화 통계검정(본편 외 보관)
```

## 두 단계 내러티브 — train1 verify → train2 retrain
- **train1 (verify)**: 원본(1차) 라벨로 베이스라인 분류기 학습 → *적중 댓글을 감지할 신호가 존재하는지* 확인. 러닝커브가 우상향이면 "신호는 있는데 데이터가 부족" → 보강 동기.
- **train2 (retrain)**: 2차 라벨링으로 C2·C3(적중) 표본을 보강한 `enriched` 학습셋으로 재학습 → 희소클래스 문제를 줄이고 *감지 성능*을 끌어올림. 평가는 자연분포 유지.

## 누수 방어 (신뢰성)
- **교차종목**: TSLA 학습 → **NVDA 평가** (다른 종목으로 일반화 확인).
- **user-disjoint**: 학습 유저는 평가셋에서 제외 (말투 암기 방지).
- **라벨 누수 0**: 적중 판정은 주가로 사후 계산, LLM은 미래를 못 봄.

## 데이터·보안
- 공개: `data/labeled/`, `data/unlabeled/splits/` — **HMAC 닉네임_ID만**(실제 닉네임 없음).
- 비공개: `data/unlabeled/raw/`(실제 닉네임·원문 대용량) 및 `.env`(API 키)는 커밋 제외.

## 현재 상태
`PROGRESS.md` 참고. 분류기 macro-F1이 다수클래스(0.231)·TF-IDF(0.323) 베이스라인을 상회 — 텍스트에 감지 신호 존재. C2 러닝커브 우상향 → 2차 보강으로 대응 중.
