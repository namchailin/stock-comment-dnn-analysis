# 2차 라벨링 — 일관성(운 vs 실력) 검증용

> 1차 라벨링([[labeling.md]])의 한계를 보완하는 **타깃 라벨링** 규칙.
> 목적: ① 운/실력을 held-out으로 검증 ② 희소클래스(C2·C3) 학습데이터 보강.

## 1. 왜 2차가 필요한가
1차는 **고변동 ~3주 윈도우**만 라벨링 → 운/실력 검증이 구조적으로 막힘:
- 종목×주차 블록이 6개뿐 → 자기상관(같은 주 예측이 같이 적중) 통제 불가
- 유저당 예측 6~12건 → split-half 검정력 부족

→ **유저당 예측 수를 늘리고, 긴 기간으로 펴면** 풀린다.

## 2. 핵심 설계 — Held-out 실력검정
1차 라벨에서 **적중 고수 후보를 선정**하고, 그들의 **다른(미라벨) 댓글을 추가 라벨링**해 **out-of-sample**로 검증한다.

```
1차 윈도우에서 C2/C3 많은 유저 선정  (in-sample, 선택)
        │
        ▼  그 유저들의 6개월 전체 댓글 중 미라벨분 추가 라벨링
새 댓글(다른 시점)에서도 적중률 높은가  (out-of-sample, 검증)
        │
   ┌────┴────┐
 높음→실력   평균회귀→운
```
- **선택은 과거 적중, 검증은 새 데이터** → 자기상관·순환 함정 없음. (회귀-평균이 곧 검정 포인트)

## 3. 샘플링 규칙
| 그룹 | 정의 | 인원 | 추가 라벨 |
|---|---|---|---|
| **고수 후보** | 1차에서 **C2/C3 ≥ 2건** | 130명 | 13,420 |
| **대조군** | 1차에서 **예측 ≥2건 & C2/C3 ≤1** | 130명(323중 무작위) | ~6,500 |

- 대상 댓글: 해당 유저의 `merged_comments` 전체(6개월) 중 **4자 이상 & 1차 미라벨**.
- 두 그룹을 **같은 기간 풀**에서 뽑아 시장효과를 비교로 통제.

### 3-1. 실현 인원 — 고수후보 130 → 119 (11명 자연탈락)
검증에 쓰는 건 **"1차에 안 쓴 새 댓글"**뿐이라(§2), 그게 0건이면 검증 불가 → 자동 탈락.
| 단계 | 고수후보 | 비고 |
|---|---|---|
| 선정(1차 C2/C3≥2) | 130 | 모집단 |
| **round-2 댓글 보유** | **119** | 11명은 6개월 댓글이 **전부 1차에서 이미 라벨** → 미라벨분 0 |
| └ 예측(C1/2/3) 1회↑ | 102 | |
| └ **예측 10회↑** (순위 진입) | **62** | `consistency_ranking.py` MIN_PRED=10 |

- **11명 탈락은 버그·누락이 아님**: 댓글 수가 적어(2~19건) 그 전부가 1차 윈도우에 걸려버린 케이스. 가진 댓글을 전부 *선정*에 써버려 *검증*용 차집합이 빈 것 → in-sample 오염 없이 빼는 게 옳음.
- (대조군은 130 선정 → round-2 보유 116)

### 3-2. 두 "검증"은 별개 트랙 (혼동 주의)
"검증=NVDA"는 **분류기** 얘기고, 실력검정은 **시점**으로 나뉜 별개 검증이다.
| | ① 분류기 교차종목 | ② 유저 실력 held-out |
|---|---|---|
| 검증 대상 | 모델(macro-F1) | 사람(적중 재현성) |
| 분리 축 | **종목** (TSLA학습→NVDA평가) | **시점** (1차선정→round-2검증) |
| 종목 범위 | NVDA only | TSLA+NVDA 섞임 |
- ②가 종목을 안 묶고 round-2 전체를 쓰는 이유: **종목 일반화가 아니라 사람의 적중 재현성**을 보는 것이라, 그 사람의 모든 out-of-sample 예측(종목 무관)을 합쳐야 검정력이 산다. NVDA-only로 좁히면 고수후보 예측 10회↑가 몇 명 안 남아 검정 불가 → **현행(round-2 전체) 유지**.

## 4. 실행 파라미터
- 모델: `qwen3.5-397b-a17b` (1차와 동일, [[labeling.md]] §7)
- 프롬프트: 동일(`prompt.py` + `domain_grounding.md`)
- **workers = 24 (절대 상향 금지)** — 고동시성(100워커)은 빈응답+재시도 과금 폭증 유발
- 러너: `run_label.py` (resume → 1차 라벨 자동 스킵)
- 라벨 후: `finalize_labels.py`로 Class 0~3 부여(주가 윈도우는 6개월 커버)

## 5. 분석 (라벨 후)
1. **Held-out 실력검정**: 고수후보 vs 대조군의 **새 댓글 적중률** 비교(종목×기간 통제) → 운/실력 판정
2. **뱃지 교차**: 고수후보 뱃지보유율(13%) vs 전체(3.9%) — 외부 고수표식 정합(sanity)
3. **분류기 재학습**: 늘어난 C2/C3로 학습 → macro-F1 개선 확인(러닝커브 데이터부족 해소)

## 6. 비용 / 주의
- 비용: 린(고수130+대조130) ≈ **$50** / 풀(고수130+대조323) ≈ $74. 재학습은 코랩=무료.
- ⚠️ **분류기 평가 누수 주의**: 보강 데이터는 *학습용*. 평가셋은 자연분포 유지하고 고수후보 편향이 test로 새지 않게 분할.
- ⚠️ **선택편향 = 검정의 일부**: 고수후보는 과거 적중으로 뽑혀 in-sample 적중률이 높은 게 당연 → **반드시 out-of-sample(새 댓글)로만** 실력 판정.
