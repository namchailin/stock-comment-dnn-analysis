# 라벨링 정책 (최종 확정)

> LLM 기반 4-Class pseudo-labeling의 확정 정책. plan.md 2절·Step3을 구현 기준으로 고정한 문서.
> 코드: `labeling/` (`prompt.py`, `domain_grounding.md`, `price_context.py`, `run_label.py`, `finalize_labels.py`)

---

## 0. 역할 분담 (핵심 — 누수 차단의 근간)
| 주체 | 하는 일 |
|---|---|
| **LLM** | 댓글 텍스트에서 **추출만**: 시점관계(예측/리액션)·예측_방향·예측_날짜·확신도·근거유형·Class0사유 |
| **코드(우리)** | **적중 여부 결정적 계산**: 방향 실현($W_{dir}$)·날짜 적중($W_{hit}$)을 주가로 계산 + Decision Logic으로 Class 부여 |

LLM은 **"예측이 맞았는지"를 절대 판정하지 않는다.** 그건 미래 결과에 의존하므로 코드가 주가 데이터로만 계산한다. → 순환논리·데이터 누수 차단.

## 1. 4-Class 정의
| 라벨 | 정의 |
|---|---|
| **0 예측없음** | 방향성 예측 없음 / 확신도<τ / 리액션(이미 움직인 것에 대한 사후반응) |
| **1 예측실패** | 방향 예측했으나 $W_{dir}$에서 미실현(반대 또는 무변동) |
| **2 방향적중** | $W_{dir}$ 방향 실현 + (날짜 없음 또는 $W_{hit}$ 미충족) |
| **3 날짜·방향적중** | 날짜 명시 + $W_{hit}$에서 예측 방향대로 변동 |

## 2. LLM 입력·출력
- **입력:** 댓글 텍스트 + **직전 가격맥락**(`price_context.py`: 작성시각 직전 48h 시간별 봉, 부족하면 직전 8봉, **미래 구간 제외**) + 도메인 그라운딩(아래) + 주주여부.
- **출력(JSON 6컬럼):** `시점관계`(예측/리액션) · `예측_방향`(상승/하락/없음) · `예측_날짜`(YYYY-MM-DD or null, 상대표현은 작성일 기준 환산) · `확신도`(0~1) · `근거_유형` · `Class0_사유`(Class0일 때 5범주 중 1).

## 3. 도메인 그라운딩 — 종목 범용 용어집 (`domain_grounding.md`)
- **종목 범용성(중요):** 용어집은 **특정 종목 애칭·인물·테마(천슬라·젠슨황·머스크 등)를 일절 넣지 않은 한국 주식 범용 용어집**으로 만들었다. 이유 — 본 분류기는 **TSLA·NVDA를 넘어 새로운 종목에도 적용 가능해야** 하므로, 종목 특화 지식을 주입하면 일반화가 깨진다. 종목 특화 md는 **주입 금지**.
- **누수 0:** 용어집은 시간 무관 일반 용어(은어·매매·심리·차트·재무·거시·시점표현·초성)라 미래 정보가 없다.
- **DL 모델 미입력:** 용어집은 **라벨링 LLM에만** 주는 보조자료이고, 학습되는 딥러닝 모델 입력에는 들어가지 않는다(모델 입력=텍스트+주주여부). → 용어집이 풍부해도 배포 모델의 종목·시즌 독립성은 안 깨진다.
- **색상 관례🚨:** 빨강=상승 / 파랑=하락(한국식, 미국 반대). 영어권 학습 LLM 최빈 실수라 명시 고정.
- INJECT 블록만 프롬프트 주입(토큰 절약), 그 아래 부록은 사람 참고용.

## 4. 누수 경계 (3중 차단)
1. **결과 미입력** — LLM엔 직전 48h만, 작성일 이후 구간 0.
2. **웹검색 OFF** — 라벨링 중 검색 금지(검색하면 그날 실제 등락을 끌어와 누수). 웹검색은 *용어집 사전 제작 1회*만 허용.
3. **적중은 코드** — Class 2·3(맞았나?)은 LLM 의견이 아니라 주가 기반 결정적 계산.
4. (보강) 프롬프트에 **hindsight 금지** 명시 — 파라메트릭 기억으로도 "이후 결과" 사용 금지.

## 5. 결정적 계산 (`finalize_labels.py`)
- **변동 판정선:** 일별 수익률 μ·σ(분석창 기준). 구간 거래일수 $h$에 대해 누적수익률 $R_W$가 $\mu h + k\sigma\sqrt h$ 이상→상승, $\mu h - k\sigma\sqrt h$ 이하→하락, 사이→무변동. **k=1.0**(EDA 확정, h=5 맥락).
- **앵커 세션 A:** 댓글 작성시각 기준 **마지막으로 마감된 미국 거래세션**(세션마감 16:00 ET ≤ 작성시각). 미국장이 KST 자정을 넘기므로 세션-날짜 귀속을 ET 기준으로 정확히 처리.
- **$W_{dir}$(방향 실현):** baseline=Close[A]. 날짜 미명시 → 끝=A+3거래일. 날짜 명시 D(단 D는 A 이후 세션, 과거날짜는 무효처리) → 끝=session(D)+3거래일. $R_{W_{dir}}$=Close[끝]/Close[A]−1, $h$=끝−A.
- **$W_{hit}$(날짜 적중, 날짜 명시만):** session(D)±1 구간. baseline=Close[D−2세션]. $h$=3. 예측 방향대로 판정선 넘으면 적중.
- 데이터 부족(평가구간 끝 등)으로 미래봉이 없으면 **보수적으로 미실현 처리**(평가가능 플래그 기록).

## 6. Decision Logic (위에서부터, τ=0.5)
```
if 예측_방향=='없음' or 확신도<0.5 or 시점관계=='리액션':   → Class 0
elif not 방향실현(W_dir):                                  → Class 1
elif 예측_날짜 없음/무효 or not 날짜적중(W_hit):            → Class 2
else:                                                      → Class 3
```

## 6.5 범위 결정 — Fold A 단방향 (비용)
LLM 라벨링 단가가 ~$0.002/콜로 예상보다 높아, **Fold A 단방향만 진행**한다: **train_A_TSLA(~20k, TSLA학습) + test_A_NVDA(~9k, NVDA평가)** 만 라벨링. train_B_NVDA·test_B_TSLA(양방향 B)는 **스킵**. → 종목 교차 일반화는 **단방향(TSLA→NVDA)** 으로만 보고하고, 양방향 교차는 한계/후속과제로 명시. 모델 학습(`model/train_colab.py`)도 Fold A 단방향(벤치마크 4모델 + 베이스라인)으로 구성.

## 7. 라벨링 모델
- **확정: `qwen3.5-397b-a17b`** (엔드포인트 최신·최대 Qwen, MoE 17B활성). OpenAI 호환 엔드포인트(`api.ssunlp.co.kr`), `temperature=0`(재현성).
- **검증 통과:** 색상관례 정확(빨간불=상승/파란불=하락), 예측vs리액션 분리, JSON 100%·에러0, 날짜환산("연말"→12-31).
- 비교 후보(필요시): GPT-5/4.1·Claude Sonnet 4.6·Gemini 2.5 — 현재 Qwen으로 충분 판단.

## 8. 검수 (★1 방어)
- 학습 전 **무작위 200~300건 사람 검수 + Cohen's κ** 보고. pseudo-label 노이즈 인정하고 모델 성능을 노이즈 상한과 함께 해석.
