## fig1 — 종목별 주가 추이 + 유의미 변동 사건
파일: `results/figures/fig1_price_story_TSLA.png`, `..._NVDA.png`
코드: `exploration/eda.py` → `fig_price_story()`, `_episodes()`

### 구성 (2단)
- **1단 (종가):** 종가 곡선 + 연속 사건 구간을 **굵은 밴드**(빨강=상승/파랑=하락)로 묶고, `▲+9% 11/05~11/17` 식 **기간·누적% 라벨**(큰 사건 최대 8개).
- **2단 (5일 누적):** 최근 $h{=}5$거래일 누적수익률 $R_W$ **막대**(사건=빨강/파랑, 무변동=회색) + 상승/하락 임계 점선. 막대 비는 구간 = 비거래일(주말·미국 공휴일).

### 변동 사건 판정·시각화 로직 (6단계)
1. **다중일 누적수익률:** 각 거래일 $t$의 $R_W(t)=\text{Close}[t]/\text{Close}[t-h]-1$ (h=5). 하루 스파이크성 단기 반전 자동 완화.
2. **임계값 (√h 스케일링):** 6개월 일일수익률 $\mu,\sigma$에서 상승 $=\mu h+k\sigma\sqrt{h}$, 하락 $=\mu h-k\sigma\sqrt{h}$ (k=1.0). 일일수익률 독립 가정상 h일 누적의 표준편차가 $\sigma\sqrt h$ 라, √h로 키워야 "1σ 유의성"이 기간과 무관하게 일정.
3. **3상태 분류:** R_W ≥ 상승임계 → +1, ≤ 하락임계 → −1, 그 외 → 0.
4. **에피소드 추출 + 짧은 틈 병합(`_episodes`):** 같은 부호 연속일을 한 사건으로 묶되, 사이의 무변동(0)이 **≤2일이면 병합**. → 잘게 쪼개진 잔밴드를 굵은 사건 몇 개로 정리.
5. **룩백 보정:** $R_W$는 trailing이라 신호가 켜질 땐 이미 상승 끝물 → 밴드 시작을 **h일 앞으로 당겨** 실제 가격이 움직이기 시작한 지점부터 칠하고, 라벨 누적%도 그 시작 종가 기준으로 계산.
6. **라벨 선별:** |누적%| 큰 순 상위 8개만 라벨 → 날짜순 정렬 후 **높이 2단 교차**(겹침 방지). 1일 사건은 양옆 반나절 패딩으로 가시화.

### 임계값 (h=5, k=1.0)
| 종목 | μ/σ(일일) | 상승임계(5일 누적) | 하락임계 |
|---|---|---|---|
| TSLA | −0.11% / 2.66% | ≥ +5.4% | ≤ −6.5% |
| NVDA | +0.17% / 2.21% | ≥ +5.8% | ≤ −4.1% |

### plan 대비 (출처 구분)
- **plan 지시:** 다중일 누적 $R_W$ vs $\mu h\pm\sigma\sqrt h$, "기준 넘긴 구간 음영(band)", 하단 $R_W$+$\pm\sigma\sqrt h$ 패널.
- **plan에 없는 추가(가독성):** 짧은 틈 병합(④), 룩백 보정(⑤), 기간·누적% 라벨+상위 8개 선별(⑥), 라벨 높이 교차, 반나절 패딩.
- **plan과 차이:** plan은 "종가·**거래량**" 포함을 명시했으나, **거래량 패널은 제거**(사용자 요청).

## fig2 — 변동 판정선 ($\mu h \pm k\sigma\sqrt{h}$)의 (k, h) Threshold Sweep
파일: `results/figures/fig2_threshold_sweep.png`
코드: `exploration/eda.py` → `fig_threshold_sweep()`, `_hk_curves()`

### 목적 (왜 그렸나)
라벨 생성 기준인 **누적기간 $h$**와 **판정폭 계수 $k$**(σ배수)를 데이터로 확정하기 위함. 핵심 개념: **변동 사건은 모델의 예측 대상이 아니라 "라벨을 만드는 자(尺)"** 다 (plan 2절). LLM이 댓글에서 뽑은 예측(방향·날짜)이 *맞았는지*를 우리가 주가로 채점하는 기준이 곧 이 변동 판정선이다. 따라서 이 그림은 *예측 정확도*가 아니라 **라벨 생성 기준의 품질**을 본다.

### 구성 (2×2)
- 행 = $k$(1.0 채택 / 1.5 엄격), 열 = TSLA / NVDA. x축 = $h \in \{2,3,4,5,6,7,8,10\}$.
- **주황(왼축) = 변동 지속율**(↑, 높을수록 좋음), **파랑(오른축) = 유의미한 변동일 비율**(≳, 너무 적지 않아야).
- 파란 점 위에 **절대 사건 수 $n$** 표기, **$n<10$은 빨강 강조**(소표본 → 지속율이 노이즈라는 경고).
- 위 행(k=1.0)에만 "h=5 채택" 표시(점선 없이 라벨).

### 두 지표 정의 (`_hk_curves`)
- **사건 판정:** trailing $R_W[i]=\text{Close}[i]/\text{Close}[i-h]-1$ 이 상승 $\mu h+k\sigma\sqrt h$ 이상 / 하락 $\mu h-k\sigma\sqrt h$ 이하.
- **변동 지속율** = 사건일 $i$ 중 **사후 $h$일 수익률** $\text{Close}[i+h]/\text{Close}[i]-1$ **의 부호가 판정 방향과 같은** 비율(follow-through). ⚠️ **"구간 내내 방향 유지"가 아니라 시작·끝 두 시점의 순(net) 방향만** 비교(끝점 부호). 높을수록 진짜 방향 사건(휩쏘 아님) → **라벨 품질**.
- **유의미한 변동일 비율** = 사건일 수 / 창 거래일 수. **사건 수 $n$ = 그 분자** = 예측을 채점할 사건이 충분한가 → **라벨 수량**.

### 발견 (실측, h=5)
| | 사건 수 $n$ | 지속율 | 변동일 비율 |
|---|---|---|---|
| TSLA k=1.0 | **21** | 57.1% | 16.8% |
| TSLA k=1.5 | **5** | 60.0% (노이즈) | 4.0% |
| NVDA k=1.0 | **41** | 41.5% | 33.1% |
| NVDA k=1.5 | **19** | **36.8% (↓)** | 15.3% |

- **$h$ 효과:** 짧으면 휩쏘가 사건으로 잡혀 사건↑·지속율↓, 길면 추세만 남아 사건↓. **h=5에서 지속율 충분 + 사건 수 적당**.
- **$k=1.5$의 함정:** 지속율을 **안정적으로 못 올림**(NVDA는 오히려 41.5→36.8% 하락). TSLA의 60%는 $n=5$ 소표본 노이즈(h=8·10의 66~75%도 $n=2{\sim}4$ 허수). 동시에 사건이 1/2~1/4로 급감(TSLA 21→5).
- **핵심 연결:** k=1.5면 변동 사건이 급감 → 그 사건 근처 댓글만 "적중" 판정 가능 → **Class 2·3(적중 고수) 라벨이 거의 안 생김** → 모델이 배울 양성 표본 소멸. 즉 "지속율 trade-off"가 아니라 **"고수 라벨 수량 붕괴"가 k=1.0을 택하는 진짜 이유**.

### 채택: **h = 5, k = 1.0σ**

### 설계 참고
- 라벨 생성은 이 확정값(**h=5, k=1.0σ**)으로 **1회 실행**(plan 지시).
- 발표/문서엔 지속율을 **"사건 후 $h$거래일 뒤까지 같은 방향 follow-through 비율"**로 표기 — *"구간 내내 유지"는 틀린 표현*.
- 사건 수 $n$ = 고수(Class 2·3) 라벨의 **상한선**. k를 키우면 고수 라벨이 부족해져 클래스 불균형 심화.
- 패널의 $n<10$ 지점 지표(특히 지속율)는 **소표본 노이즈** → 신뢰 금지.
- (옛 fig2 `fig_threshold_sweep` 막대형은 가독성 문제로 폐기, 현 함수가 그 이름·파일명 승계.)

## fig3 — 댓글 토큰 수 분포 (max_length 결정 근거)
파일: `results/figures/fig3_length_dist.png`
코드: `exploration/eda.py` → `fig_length()`, `_token_lengths()`
부속 산출물: `results/over512_tokens.md` (512 초과 댓글 전문, 종류별)

### 목적
딥러닝 입력 **max_length**를 분포 기반으로 결정. 초과분은 **Truncation(뒷부분 잘라냄)**, 짧은 글은 **Padding(빈자리 채움)**으로 길이를 맞춘다 (둘 다 "제거"가 아니라 길이 정렬 전처리; plan 3-1).

### 구성 (broken axis)
- **선형 x + 축 단절(`//` 사선, matplotlib 공식 레시피):** 좌 0~80토큰 세밀 / 우 80~최대 압축. 선형 비례를 지키면서 긴 꼬리까지 표시.
- 분위 점선·가로 라벨: 최소 3 · 중앙값 11 · P95 43 · P99 222 · 최대 1339 (개, **[CLS]/[SEP] 포함**).
- **512 초과 = 투명 빨강 박스(얇은 실선 테두리) + 빨강 글씨 "Max Token 512개 초과 (728건, 0.31%)"** — 512는 KcELECTRA 한계.

### 토큰화 설정 (확정)
- 토크나이저 **KcELECTRA `beomi/KcELECTRA-base`** (byte-level BPE, vocab 30000, model_max 512).
- URL 제거(`https?://\S+|www\.\S+`) 후, 토큰 수 = **내용 토큰 + 2([CLS],[SEP])**. 캐시 `data/token_len_kcelectra.npy`.

### 조사 과정 (시행착오 포함)
1. **글자 수**로 시작 → 로그축 히스토그램·CDF+커버율·broken axis 등 시안 비교 → **broken axis** 채택(선형 비례 + 긴 꼬리 + 관례적 단절).
2. **토큰화 전환:** 글자 수 → 모델 토큰 수로 전환. 토크나이저 **KcELECTRA `beomi/KcELECTRA-base`** 확정(byte-level BPE, 한계 512토큰). byte-BPE는 한국어·반복 문자를 큰 단위로 묶어 토큰 수가 적게 나온다(예: `테/슬라`).
3. **노이즈 점검:** URL 1,633건 포함, 순수 URL 392건은 제거 후 빈 글 되어 제외. **이미지는 데이터에 없음** — 크롤러가 텍스트(title+message)만 수집, 이미지-only 댓글은 수집 단계 `is_meaningful`(빈글·3자 이하·이모티콘만 제거)로 드롭되어 흔적·양 측정 불가.
4. **KcELECTRA 특수토큰 함정(중요):** 이 토크나이저는 `post_processor=None`이라 `add_special_tokens=True`로도 [CLS]/[SEP]가 **안 붙음**(min=1) → **+2 보정**으로 통일. → 메모리 `kcelectra-special-tokens`.

### 발견·수치 (KcELECTRA, [CLS]/[SEP] 포함, N=235,611)
| 지표 | 토큰 수 | (참고: 글자 수) |
|---|---|---|
| 최소 / 중앙 / 평균 | 3 / 11 / 19.9 | 4 / 17 / 34.6 |
| P95 / P99 / 최대 | 43 / 222 / 1339 | 82 / 425 / 2051 |

| max_length | 초과(잘림) 건수·비율 |
|---|---|
| 64 | 7,026 (2.98%) |
| 128 | 3,716 (1.58%) |
| 256 | 2,005 (0.85%) |
| **512** (KcELECTRA 한계) | **728 (0.31%)** |

- 롱테일(평균 19.9 > 중앙 11). 대부분 ~10~40토큰.
- **512 초과 728건 = 거의 전부 정보성 장문**(한글 뉴스·분석 720, 영문 리포트 8). 전문은 `results/over512_tokens.md`.
- byte-BPE가 반복 문자(점 도배 등)를 강하게 압축해 **도배성 장문조차 512 미만**으로 들어옴. 즉 512 초과(잘리는) 건은 노이즈가 아니라 **정상 정보성 장문**.

### 설계 참고
- **max_length 후보별 커버:** 128 → 98.4%(1.58% 잘림), 256 → 99.2%, **512 → 99.7%(0.31%만 잘림)**. 길이↑ = $O(L^2)$ 연산·패딩 낭비 vs 커버 trade-off.
- 잘리는 건 **정보성 장문의 뒷부분** → 결론·목표가가 뒤에 있으면 정보 손실 가능. truncation 방향(기본=뒤 자르기) 영향 감안.
- **KcELECTRA 학습 시 [CLS]/[SEP] post_processor 반드시 세팅**(없으면 [CLS] 누락 → 에러 없이 조용한 성능 저하). 벤치마크 4종이면 모델별 특수토큰 포함 여부 1회씩 점검. → 메모리 `kcelectra-special-tokens`.
- 토큰 수·max_length는 **특수토큰 +2 포함** 동일 기준.
- URL 제거·3자 이하·이모티콘만 제거는 **크롤러 `is_meaningful`(수집 단계)에서 이미 수행** → preprocess에서 재현 필요.

## fig4 — 유저 메타데이터 분포 (주주 / 뱃지)
파일: `results/figures/fig4_user_meta.png`(TSLA+NVDA), `..._TSLA.png`, `..._NVDA.png`
코드: `exploration/eda.py` → `fig_user_meta()`, `_user_meta_panel()`

### 구성 (3패널, 종목별 3파일)
- **파이1 유저 내 주주 점유율** / **파이2 유저 내 뱃지 보유자 점유율** / **막대 뱃지 종류별 유저 수**(막대 끝에 유저수·비율, 비율=뱃지 보유자 내).

### 집계 방식 (사실 — 측정 정의)
- **유저(`닉네임_ID`) 단위 집계**로 댓글 수 편향 제거. 한 유저가 N개 댓글을 써도 1명으로 계산.
- 주주 = 그 유저 댓글 중 `주주_여부`가 한 번이라도 True면 주주(`max`). 뱃지 보유 = `has_badge` `max`. 대표 뱃지 = 그 유저가 가장 많이 단 뱃지(`mode`).

### 검증 (독립 재계산, 이상 없음)
- 원본 236,003행, **commentId 중복 0**, 작성일 파싱 실패 0(제거 행 없음). 유저 **63,040명**.
- 종목별 유저 합 = TSLA 43,868 + NVDA 24,138 = 68,006. **양쪽 모두 댓글 단 유저 4,966명**(중복) → **합집합 63,040 = 전체와 일치**. (combined 파일은 합산이 아니라 `nunique`로 올바르게 중복 제거)

### 핵심 수치 (유저 기준)
| | 유저 수 | 주주 | 뱃지 보유자 |
|---|---|---|---|
| 전체 | 63,040 | 66.1% | 3.9% |
| TSLA | 43,868 | 64.6% | 3.1% |
| NVDA | 24,138 | 65.7% | 6.7% |

- **댓글 기준 ≠ 유저 기준(사실):** 뱃지 댓글 기준 10.1% → 유저 기준 3.9% (뱃지 보유자가 1인당 댓글을 더 많이 씀). 주주 67.9% → 66.1%.
- **뱃지는 6종**("Top 10" 아님): 1억대 자산가 · 수익금 상위 5%/1% · 팔로워 부자 · 5억대 · 10억대 자산가. 종류별 유저 수는 각 파일 막대 참조(전체 기준 1억대 자산가 1,588명(64.1%)이 최다).
- 종목별 뱃지 종류 수(전체 기준, 대표뱃지): 자산가류 1,003(TSLA)/1,117(NVDA), 수익금류 232/368, 팔로워 132/135.

### 관측 (데이터가 보증하는 진술만)
- 주주 비율은 두 종목 모두 **~65%로 유사**(64.6% vs 65.7%).
- 뱃지 보유자 비율은 **종목별로 다름**(TSLA 3.1% vs NVDA 6.7%).
- (참고) 각 종목 분석 창 등락: TSLA −19.1%(2025-10-01→2026-03-31), NVDA +17.4%(2025-12-01→2026-05-31). **단 두 창은 같은 기간이 아님**(서로 다른 6개월).
- ⚠️ **인과·행동 해석 금지**: "왜 뱃지 비율이 다른가/주주가 비슷한가"는 이 단면 데이터로 단정 불가(선택효과 vs 인과 구분 불가, 이탈은 종단 데이터 필요). 뱃지가 종목별 성과인지 계정 전체 속성인지도 **미확인** → 확인 전 수익률과 연결 금지.

### 설계 참고
- 비율은 **반드시 유저 단위**로 낼 것(댓글 단위는 활동량 많은 유저로 과대편향 — 뱃지 10.1%→3.9% 사례).
- 주주/뱃지는 **모델 입력 피처로 쓰지 않음**(plan: 텍스트만 입력). 해석·검증용 보조 지표.
- 뱃지 정의(종목별 vs 계정 전체) **미확인** — 라벨/해석에 쓰기 전 확인 필요.
- 종목별 분석 창이 다르므로(기간 비동일) 종목 간 직접 비교 시 주의.

## fig5 — 주가 변동성과 댓글량의 상관관계  (소제목: 고변동 구간 분석 정당화)
파일: `results/figures/fig5_volatility_timeseries.png`
코드: `exploration/eda.py` → `fig_volatility_vs_volume()`, `_daily_merge()`

### 구성
- 2패널(TSLA/NVDA) 이중축 시계열: **보라 선 = |일일 등락률|(%)**, **회색 막대 = 일일 댓글량**. 각 패널 제목에 상관 통계(거래일 n, Pearson r, Spearman ρ, p).

### 집계 방식 (사실 — 측정 정의)
- `_daily_merge`: 주가 일봉(거래일)에 댓글 수를 일자별로 붙임. 상관은 **거래일 단위**(x=|등락률|, y=댓글량).
- **비거래일(주말·공휴일) 댓글은 다음 거래일(개장일)로 이월 합산** → 누락 0%(이전엔 TSLA 10.6%/NVDA 4.8% 드롭됐었음). 단 개장일 댓글량이 **누적 효과로 높을 수 있음**(그림에 명시).

### 결과 (실측)
| 종목 | n(거래일) | Pearson r (p) | Spearman ρ (p) | R²(≈r²) |
|---|---|---|---|---|
| TSLA | 125 | 0.45 (p<.001) | 0.42 (p<.001) | ≈0.20 |
| NVDA | 124 | 0.33 (p<.001) | 0.36 (p<.001) | ≈0.11 |

- **Spearman ≈ Pearson** → 이상치·치우침에 견고(순위 기반으로도 동일 결론). 둘 다 **p<.001**, 거래일 n=124~125라 우연 가능성 사실상 없음.

### 관찰·함의
- **관찰:** 일일 변동성이 클수록 그날 댓글량이 **유의하게 많다**(양의 중간 상관, p<.001).
- **함의 1 — 데이터 적합성:** 주가 사건(변동)과 커뮤니티 활동(댓글)이 실제로 맞물려 있다. 변동이 큰 시점에 사람들이 그 종목을 활발히 말한다 → 댓글 안에 주가 관련 발언(예측·반응)이 담겨 있을 개연성이 높다 → **"댓글을 분석해 주가 예측 언어를 잡아내자"는 접근이 헛다리가 아니다.**
- **함의 2 — 분석 대상 선택의 타당성:** 변동성 큰 종목·기간일수록 댓글이 풍부하니, 변동성 높은 데이터(테슬라·엔비디아)를 다루는 본 연구의 **데이터 선택이 합리적**(신호도 많고 표본도 많음).

### 설계 참고 (선 긋기)
- 이 차트의 역할은 **문제 정의 뒷받침**이지 정밀 예측이 아님 — 중간 상관(r 0.33~0.45)이면 논지 충분.
- 단일 요인(변동성)이 댓글량 분산의 **약 11~20%(R²)** 설명; 나머지는 실적발표·뉴스·요일효과·전반적 화제성 등 → 한 요인치고 꽤 됨.
- TSLA>NVDA 차이는 직관(개인 화제성·변동성 반응이 더 격함)과 일치하나 **인과 단정 금지**.

### fig5가 정당화하지 "않는" 것 (방어용)
- ❌ **"이 6개월 구간을 골랐다"는 구간 선정 행위** — 그건 종목·기간 선정(1절)의 근거지 fig5가 아니다.
- ❌ **인과("변동성이 댓글을 유발한다")** — fig5는 **상관**일 뿐. 토스의 등락 표시(빨강/파랑 %) 같은 **제3요인**이 변동성·댓글을 동시에 끌어올릴 수 있다. 방향성 주장 금지.
